## 출력 파일 명세

- **파일명**: `investment_dashboard.html` (고정, 변경 금지)
- **저장 위치**: Skills.md 폴더와 동일한 루트 디렉토리
- **형식**: 단일 HTML 파일 (CSS/JS 인라인 포함, 외부 파일 분리 금지)
- **외부 의존성**: 아래 CDN만 허용

```html
<script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js"></script>
```

- **데이터 로드**: 모든 CSV/JSON 데이터를 HTML `<script>` 태그 안에
  raw 텍스트로 직접 삽입 후 브라우저 JS에서 파싱
  - ❌ fetch() 금지
  - ❌ Python·Bash로 데이터 **변환·집계·계산** 금지 (KPI·매핑·필터링 모두 JS에서)
  - ✅ `build_dashboard.py` **단일 파일**의 raw 텍스트 인라인 어셈블리는 허용
    (원본 파일 읽기 → 템플릿 치환 → 저장만 수행, 데이터 내용은 변경하지 않음)
- **CSV 파싱 방식**: split('\n') → split(',') → trim() → BOM 제거
- **JSON 파싱 방식**: JSON.parse() 직접 사용
- **반응형**: 불필요 (데스크탑 1280px 이상 기준)
- **차트 컨테이너 규칙 (필수)**: 모든 `<canvas>`는 **`position:relative` + 고정 `height`(px)** 인 부모 div로 감싼다.
  Chart.js 옵션 `responsive:true` + `maintainAspectRatio:false` 조합은 부모 높이를 측정하므로,
  부모에 높이가 없으면 매 리사이즈마다 캔버스가 무한히 늘어난다.
  - ✅ `<div style="position:relative;height:240px"><canvas id="x"></canvas></div>`
  - ❌ `<canvas id="x" height="240"></canvas>` (responsive 모드에서 height 속성은 무시됨)
  - 기본 권장 높이: 라인/바차트 240px, 컴팩트 차트 200px, sparkline 50px

## Step 1: UI 컴포넌트 — 드롭다운 + 기간 토글

### 기간 토글 (탭별 분리 배치)

**핵심 원칙**: 토글은 **기간 의존 항목 바로 옆**에 위치한다. 영향 범위가 좁은 뷰에서 헤더 글로벌 토글을 그대로 두면 사용자가 "다른 카드도 같이 바뀌어야 하나?"라고 오해할 수 있으므로, 뷰별로 토글 배치를 분리한다.

- 옵션: **YTD / 1M / 3M / 1Y** (4개 토글 버튼)
- 기본 선택: **YTD** (`2-skills-criteria.md` 기준)
- 활성 토글 색상: `#3A9E9E` 배경 + 흰 텍스트
- 비활성: 투명 배경 + `#8B949E` 텍스트
- 두 토글은 동일한 `state.period`를 공유 — 한쪽에서 변경 시 다른 쪽 active 상태도 동기화

| 뷰 | 토글 위치 | 토글 변경 시 재렌더링 범위 |
|---|---|---|
| **PM** | 헤더 우측 (날짜 뱃지 옆), `id="period-toggle-global"` | 기간 의존 항목이 다수(KPI/라인차트/테이블/배너) → PM 뷰 **전체 재렌더링** |
| **RA** | "주가 추이 vs 목표주가" 차트 카드 **내부 우측 상단** (제목과 같은 줄), `id="period-toggle-ra"` | RA에서 기간 의존은 주가 차트 하나뿐 → **주가 차트만 재렌더링**. 어닝 서프라이즈·분기실적·피어·재무건전성·KPI는 그대로 유지 (불필요한 재애니메이션 금지) |

**탭 전환 시 토글 가시성**:
- PM 탭 활성 → `period-toggle-global` 표시, `period-toggle-ra` 숨김
- RA 탭 활성 → `period-toggle-global` 숨김, `period-toggle-ra` 표시
- 초기 진입 시(`init()`) 한 번 동기화 함수 호출

**구현 노트**:
- 두 토글 모두 `.period-btn` 클래스 사용 → 단일 클릭 핸들러로 처리
- 클릭 시: `state.period` 갱신 → 모든 `.period-btn`의 active 상태를 `dataset.p === state.period` 기준으로 일괄 토글 → 뷰별 재렌더링 함수 호출
- 차트·KPI 제목에는 여전히 선택 기간 명시 (예: "수익률 (YTD)", "주가 추이 vs 목표주가 (1M)")

### 기간 토글 적용 범위 (필수 가시화)

토글 영향 범위가 사용자에게 모호하지 않도록, **모든 KPI·차트·패널은 기준 기간을 라벨로 명시**한다. 라벨 누락 금지.

| 분류 | 항목 | 라벨 표기 | 토글 동작 |
|---|---|---|---|
| 토글 따라감 | PM 수익률 KPI | `수익률 ({선택기간})` | 값·라벨 동시 갱신 |
| 토글 따라감 | PM 누적수익률 라인차트 제목 | `누적 수익률 vs 벤치마크 ({선택기간})` | 슬라이스·라벨 동시 갱신 |
| 토글 따라감 | RA 주가 추이 차트 제목 | `주가 추이 vs 목표주가 ({선택기간})` | 슬라이스·라벨 동시 갱신 |
| 토글 따라감 | PM 보유종목 테이블 "수익률"·"기여도" 컬럼 | 컬럼 헤더 `수익률 ({선택기간})` / `기여도 ({선택기간})` | `MD.tickers[ticker].returns[기간]` 사용, 갱신 |
| 토글 따라감 | PM 인사이트 배너의 초과수익 메시지 | "벤치마크 대비 초과수익 {값}p ({선택기간})" | 갱신 |
| 고정 (Skills 명세) | PM Sharpe Ratio | `Sharpe (1Y)` | 라벨에 `1Y` 박제, 토글 무관 |
| 고정 (Skills 명세) | PM 트래킹 에러 | `트래킹에러 (1Y)` | 라벨에 `1Y` 박제, 토글 무관 |
| 고정 (Skills 명세) | PM MDD | `MDD (전체기간)` | 라벨에 `전체기간` 박제, 토글 무관 |
| 고정 (Skills 명세) | PM 총 AUM | `총 AUM (현재)` | 현재 시점 스냅샷, 토글 무관 |
| 고정 (Skills 명세) | PM 도넛 차트 제목 | `구성종목 배분 (현재 기준)` | 현재 비중 스냅샷, 토글 무관 |
| 고정 (Skills 명세) | PM 리스크 모니터링 패널 제목 | `리스크 모니터링 (전체기간)` | 4지표 모두 전체 history 기반 |
| 고정 (Skills 명세) | PM 리밸런싱/집중도 패널 제목 | `리밸런싱 신호 (현재 기준)` / `집중도 리스크 (현재 기준)` | 현재 비중 기반 |
| 고정 (Skills 명세) | RA 펀더멘털 KPI 5개 (PER/PBR/EPS/ROE/배당) | KPI 카드 그리드 상단에 안내 1줄: `펀더멘털 지표는 최신 공시 기준` (`--mute` 색상, `--fs-sm`). 토글이 주가 차트 카드 안에 있으므로 "기간 토글은 주가 차트에만 적용" 문구는 불필요 | 토글 무관 |
| 고정 (Skills 명세) | RA 분기실적·피어·재무건전성·종합인사이트 3패널 | 각 카드 제목 옆 `(현재 기준)` 또는 `(최신 공시 기준)` 부기 | 토글 무관 |

**원칙**:
- 토글 따라가는 항목 → 라벨에 `{선택기간}` 동적 치환
- 고정 항목 → 라벨에 기준(`1Y` / `전체기간` / `현재` / `최신 공시 기준`)을 **하드코딩**해서 박제
- "왜 안 바뀌지?"라는 의심이 들지 않도록 **모든 카드·차트·패널 제목에 기간 기준이 1회 이상 노출**되어야 한다
- RA KPI 그리드 상단 안내 문구는 패널 자체가 아닌 그리드 외곽에 1회만 표시 (KPI 5개 각각에 라벨 다는 것보다 가독성 우수)

### PM 뷰 ETF 드롭다운
- 형식: `<select>` 드롭다운
- 옵션: market_data.json에서 role = "etf"인 ticker 전체 자동 로드
- 표시 이름 우선순위:
  1. fundamentals.short_name 있으면 사용
  2. etf_info.tracking_index 있으면 사용
  3. 둘 다 없으면 **ticker 코드 그대로** 표시 (한글명·약칭 임의 부여 금지)
- 기본 선택: 첫 번째 etf ticker
- 선택 변경 시: 전체 PM 뷰 데이터 즉시 재렌더링
- 표시 이름 예시 (market_data 기준, Skills에 명시된 경우에만 사용):
  - `423920.KS` → "TIGER 미국필라델피아반도체나스닥"
  - `412570.KS` → "TIGER 2차전지테마"
  - `123320.KS` → "TIGER 코스피200"
  - `243880.KS` → "TIGER 반도체"
- **주의**: 위 예시에 없는 ETF ticker의 표시명은 데이터 필드에서만 가져온다. AI가 임의로 한글명을 만들지 않는다.


### RA 뷰 종목 드롭다운
- 형식: 검색 가능한 `<select>` (검색 input + 옵션 필터링)
- 옵션: `1-skills-data.md` Step 3의 **RA 드롭다운 제외 기준(A·B·C)**을 통과한 holding만 표시
- 표시 형식: "{종목명} ({ticker})" 예: "삼성SDI (006400.KS)"
  - 이름 깨진 종목(조건 C): "{ticker} ({ticker})" 형식으로 표시
- 기본 선택: `006400.KS` (삼성SDI)
- 정렬: 종목명 가나다 순
- 선택 변경 시: 전체 RA 뷰 데이터 즉시 재렌더링
