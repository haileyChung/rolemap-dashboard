## Step 1: PM 리포트 렌더링 순서

ETF 드롭다운 선택 변경 **또는 기간 토글 변경 시** 아래 순서로 전체 재렌더링:
```
1. ETF 드롭다운 + 패시브/테마형 뱃지
2. KPI 카드 5개        수익률(기간) / 총 AUM / Sharpe(1Y) / MDD / 트래킹에러(1Y)
3. 인사이트 배너       조건 평가 → 자동 메시지 (Step 3 기준)
4. 누적 수익률 라인    ETF vs 벤치마크
5. 도넛 차트           구성종목 배분
6. 리스크 모니터링 패널  Beta / VaR 95% / 연환산 변동성 / 트래킹에러 + 30D 롤링 변동성 sparkline
7. 리밸런싱 신호 패널  패시브 → 섹터별 갭 / 테마형 → 집중도 리스크
8. 보유종목 테이블     weight_pct 내림차순 (의견·기여도·괴리 포함)
```

## Step 2: RA 리포트 렌더링 순서

종목 드롭다운 선택 변경 **또는 기간 토글 변경 시** 아래 순서로 전체 재렌더링:
(기간 토글은 시계열 차트만 슬라이스 갱신, 펀더멘털 KPI 값은 유지)
```
1. 종목 드롭다운 (검색 가능)
2. 종목 헤더              종목명·투자의견 배지·현재가·목표주가·괴리율 + 동섹터 빠른 전환 칩
3. KPI 카드 5개           PER(Fwd) / PBR / EPS(TTM) / ROE / 배당수익률
4. 인사이트 배너          조건 평가 → 자동 메시지 (Step 3 기준)
5. ROW 1 (2열):
     주가 추이 vs 목표주가 라인차트(기간 적용) | 어닝 서프라이즈 묶음 바차트
6. ROW 2 (3열):
     분기실적 바차트 | 피어 밸류에이션 테이블 | 재무 건전성 진행바
7. ROW 3 (3열):
     종합 인사이트 3패널  밸류에이션 / 실적 모멘텀 / 리스크 (`5-skills-ra.md` ROW 3)
```

## Step 3: 자동 인사이트 배너

KPI 카드 바로 아래 배너 영역에 표시.
조건 동시 충족 시 우선순위 순으로 최대 3개까지 표시.
조건 없음 시 기본 메시지 반드시 표시 (빈 배너 금지).

**메시지 본문은 `references/insight-templates.md` §2(PM)·§3(RA)에서 가져온다.**
본 파일은 트리거 조건과 우선순위만 정의하고, 문장은 단일 출처로 분리되어 있다.

### PM 인사이트
| 우선순위 | 조건 | 색상 | 메시지 |
|---|---|---|---|
| 1 | MDD < -15% | 🔴 `#E24B4A` | "리스크 한도 초과 — 포지션 축소 검토" |
| 2 | returns.1m > 50% | 🟡 `#C8963E` | "단기 {값}% 급등 — 차익실현 검토" |
| 3 | 1위 종목 weight > 30% | 🟡 `#C8963E` | "{종목명} {값}% 집중 — 분산 검토" |
| 4 | 초과수익 > +10%p | 🟢 `#1D9E75` | "벤치마크 대비 초과수익 {값}%p" |
| 5 | 초과수익 < -10%p | 🔴 `#E24B4A` | "벤치마크 대비 하회 {값}%p" |
| — | 조건 없음 | 🟢 `#1D9E75` | "포트폴리오 정상 운용 중" |

### RA 인사이트
| 우선순위 | 조건 | 색상 | 메시지 |
|---|---|---|---|
| 1 | 괴리율 ≥ +20% | 🟢 `#1D9E75` | "목표주가 대비 {값}% 상승 여력" |
| 2 | 괴리율 ≤ -10% | 🔴 `#E24B4A` | "목표주가 하회 {값}% — 하락 리스크" |
| 3 | 최근 분기 매출 서프라이즈 ≥ +5% | 🟢 `#1D9E75` | "최근 분기 매출 컨센서스 +{값}% 상회" |
| 4 | 최근 분기 매출 서프라이즈 ≤ -5% | 🔴 `#E24B4A` | "최근 분기 매출 컨센서스 {값}% 하회 — 어닝 쇼크" |
| 5 | PER < 섹터평균 × 0.8 | 🟢 `#1D9E75` | "섹터 평균 대비 PER 저평가" |
| 6 | ROE > 15% | 🟢 `#1D9E75` | "ROE {값}% — 우수한 자본효율성" |
| 7 | 부채비율 > 200% | 🟡 `#C8963E` | "부채비율 {값}% — 재무 레버리지 주의" |
| — | 조건 없음 | 🟢 `#1D9E75` | "밸류에이션 정상 범위" |

**어닝 서프라이즈 인사이트 데이터**: `consensus_data.json → tickers[ticker].quarters[]`의
**가장 최근 분기**(배열 마지막 원소)의 `revenue_surprise_pct` 사용.
금융업(`is_financial: true`)이어도 매출 서프라이즈는 적용. 단 `consensus_data.json` 없으면 조건 3·4 평가 생략.

## Step 4: 디버그 패널 (페이지 내 매핑 상태 가시화)

`1-skills-data.md` Step 1.3 콘솔 로그와 **동일한 정보**를 페이지에서도 확인할 수 있도록
**접이식 디버그 패널**을 페이지 우하단에 고정 배치한다.

### 위치 및 동작
- 위치: `position: fixed; bottom: 16px; right: 16px;`
- 기본 상태: **접힘** (작은 토글 버튼만 표시 — "🔍 매핑 상태")
- 클릭 시: 펼쳐져서 매핑 결과·데이터 로드 상태 표시
- 다시 클릭: 접힘
- z-index: 9999 (차트 위에 떠야 함)

### 표시 내용 (펼친 상태)

```
┌─ 🔍 매핑 상태 ─────────────────[×]┐
│ 데이터 로드                       │
│   ✅ market_data.json (99 tickers)│
│   ✅ financial_index.json (10,922)│
│   ✅ sector_targets.json          │
│   ✅ consensus_data.json (91)     │
│   ✅ etf_holdings_*.csv (4 files) │
│                                   │
│ 역할 매핑 (성공)                  │
│   close_role  ← close             │
│   weight_role ← weight_pct        │
│   per_role    ← per_fwd           │
│   ...                             │
│                                   │
│ 역할 매핑 (실패 / 비활성화)       │
│   ⚠ target_price_role: 일부 종목  │
│       null (RA 패널 일부 — 표시)  │
│                                   │
│ 필터 적용 결과                    │
│   RA 제외 (조건 A): 4건           │
│   이름 깨진 종목 (조건 C): 4건    │
│                                   │
│ 알려진 이슈                       │
│   금융업 영업이익 0: 17건         │
│   해외 holding (.KS 없음): 1건    │
└───────────────────────────────────┘
```

### 데이터 소스
- 로드 상태: 각 파일 파싱 직후 카운트
- 역할 매핑: Step 1.2의 매칭 결과 그대로
- 필터 결과: `1-skills-data.md` Step 3 RA 제외 기준 적용 결과
- 알려진 이슈: `references/current-dataset.md`의 "데이터 특이사항" 섹션 직접 참조

### 스타일
- 패널 배경: `var(--card)` `#161B22`
- 토글 버튼: `var(--teal)` `#3A9E9E` 배경, 흰 텍스트
- 성공 ✅ → green / 경고 ⚠ → gold / 실패 ❌ → red
- 폰트: 11px monospace (정렬용)

### 콘솔 로그와의 관계
- 콘솔 로그(F12)는 **개발자용 / 영구 기록**
- 디버그 패널은 **사용자용 / 즉시 확인용**
- 둘 다 동일 정보를 출력 (이중화)
